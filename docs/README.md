# Documentation Index

Project

Distributed Payroll Processing Engine

---

# Purpose

This folder contains the product, architecture, technical design, decision records and operational documentation for the platform.

---

# Structure

- 00-vision: project vision and engineering philosophy.
- 01-prd: product requirements.
- 02-architecture: architecture overview and domain glossary.
- 03-tdd: technical design documents.
- 04-adr: architecture decision records.
- 05-specs: feature specifications.
- 06-testing: performance and chaos plans.
- 07-deployment: deployment documentation.
- 08-observability: monitoring and observability documentation.
- 09-tracking: implementation roadmap and task tracking.

---

# Working Rule

Before implementing a feature:

1. Read the relevant specification.
2. Check the TDD chapters.
3. Check ADRs.
4. Add a new ADR if a decision is missing.
5. Write tests before implementation.
